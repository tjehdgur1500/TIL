### 1. 네임드튜플

- 일반 튜플과 네임드 튜플의 차이

1. 튜플은 불변 객체

일반 튜플과 네임드 튜플 둘다 불변 객체이지만 확연한 차이점이 있다.
만약 두점 사이를 구하는 공식을 작성한다 했을때 에시이다.

```python
# 일반 튜플

pt1 = (1.0, 5.0)
pt2 = (2.5, 1.5)

from math import sqrt

line_leng1 = sqrt((pt2[0] - pt1[0])**2 + (pt2[1] - pt1[1])**2)

print(line_leng1)

# >> 3.8078865529319543

# 네임드 튜플 선언

from collections import namedtuple

# 네임드 튜플 사용
# 값의 이름들이 존재하고 메모리를 덜 잡아먹게 되어 효율적임

# typename , feild name 으로 세팅가능
Point = namedtuple('Point', 'x y')

pt1 = Point(1.0, 5.0)
pt2 = Point(2.5, 1.5)

line_leng2 = sqrt((pt2.x - pt1.x)**2 + (pt2.y - pt1.y)**2)
print(line_leng2)

# >> 3.8078865529319543
```

둘의 출력 결과는 분명 같지만 내부적으로 매우 다르다.
일반 튜플로 작성했을때 어떤게 x 고 y인지 정확히 알 수 가없다.
네임드 튜플로 작성했을때는 클래스처럼 이름을 줄 수 있으면서 그안에 필드네임을 줄 수 있다.
이 필드네임에 값이 자동으로 접근하는것이 네임드 튜플 특징이다.

마치 클래스 처럼 접근할 수 있다.

```python

# 네임드 튜플 선언 방법
Point1 = namedtuple('Point', ['x', 'y'])
Point2 = namedtuple('Point', 'x, y')
Point3 = namedtuple('Point', 'x y')
Point4 = namedtuple('Point', 'x y x class', rename=True)
# rename 을 True 로 하면 중복된 필드네임을 무작위로 _ 언더바를 붙여서 네이밍 해준다.

```

```python

# dict to unpacking
temp_dict = {'x': 75, 'y': 55}

# 객체 생성

p1 = Point1(x=10, y=35)
p2 = Point2(20, 40)
p3 = Point3(45, y=35)
p4 = Point4(10, 20, 30, 40)
p5 = Point3(**temp_dict)
## p5 에 temp_dict 하나만 입력한다면 x 하나라고 인식하기 때문에 ** kwargs 로 언패킹 한것이다.

print(p1, p2, p3, p4, p5)

```

정말 클래스랑 유사하게 쓰일 수 있어서 아마 클래스랑 헷갈릴 수도 있을것 같다.

- 유용한 네임드 튜플 메소드

```python

# 사용

print(p1.x + p2.y)  # 클래스 변수 접근 방식

# unpacking

x, y = p3
print(x + y)

# rename 테스트
print(p4)

# 네임드 튜플 메소드

temp = [52, 38]

# _make() : 새로운 객체 생성
p4 = Point1._make(temp)
print(p4)

# _fields() : 필드 네임 확인
print(p1._fields, p2._fields, p3._fields)

# _asdict() : orderedDict 반환 , 정렬기능

print(p1._asdict(), p4._asdict())

# _replace() :  수정된 새로운 객체 반환  / 튜플은 불변이기 때문에

print(p2._replace(y=100))


```

### 2. 리스트 컴프리헨션

리스트 컴프리헨션은 매우 간단하게 for문으로 리스트를 만들 수 있다.

```python

# 실 사용 실습
# 학생 전체 그룹 생성
# 반 20명 , 4개의 반 - > a b c d

# 네임드 튜플 선언 / 리스트보다 빠름 , 불변 객체이기 때문에

Classes = namedtuple('Classes', ['rank', 'number'])

# 그룹 리스트 선언
numbers = [str(n) for n in range(1, 21)]

number = []
for n in range(21) :
    number.append(n)

```

둘은 같은 결과이지만 이 또한 내부적으로 굉장히 다르다.
리스트 컴프리헨션으로 한줄로도 작성할 수 있다.

```python

# 그룹 리스트 선언
numbers = [str(n) for n in range(1, 21)]
ranks = 'A B C D'.split()

# List comprehension
students = [Classes(rank, number) for rank in ranks for number in numbers]
print(len(students))
print(students)

# 가독성 x , 리스트 컴프리헨션을 남용할 경우
print()
print()

students2 = [
    Classes(rank, number)
    for rank in 'A B C D'.split()
        for number in [str(n) for n in range(21)]
]

print(students2)

# 출력
print()
print()

for s in students:
    print(s)

```
