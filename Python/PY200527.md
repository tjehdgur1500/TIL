## 테스트 코드

코드를 테스트하는 방법론

- 코드를 변경하면 연쇄작용으로 관련된 모든 코드가 깨짐
- 작은 코드 수정에도 나비효과로 인해 무슨일이 벌어질지 모르므로 항상 자동화 테스트 코드를 실행
- 코드 테스트 하는 가장 쉬운 방식 `print(x)` `print()` 매우 느리고 테스트 코드가 수백/수천개가 되면 인식 불가
- 회원가입/로그인/프로필업데이트/인스타글작성/댓글작성 환경에서 프로필 `name` -> `username` 변경시 무수히 많은 코드 문제 발생

## Transeciton & Roll-back

모든 테스트코드르 작성하고 실행할때는 의존성이 없어야 한다.

개별적인 기능을 테스트 해야하기 때문이다

transetion 은 테스트가 전부 작동되는 일련의 과정을 말한다.

만약 하나라도 에러가 날시 다시 처음으로 돌아가는 것을 roll-back이라한다.
