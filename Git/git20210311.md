## Git 작동원리

![](https://upload.wikimedia.org/wikipedia/commons/thumb/e/e0/Git-logo.svg/1200px-Git-logo.svg.png)

### git

깃은 버전관리 툴이다.<br>
깃허브랑 연동해 파일들을 관리할 수 있으며 , <br>
많은사람과 협업 , 공유가 가능하다. <br><br>

회사에서도 많이 사용했었고 , git 을 사용 안하기엔 너무 편한 툴이다. <br><br>

git 은 이곳에서 다운로드 할 수 있다. <br>
https://git-scm.com/downloads

나는 git의 설치과정부터 설명하기보단 로컬에서 원격저장소 까지 가는 과정을 기록하려한다. 

git 은 아래 사진하나로 어느정도 설명이 된다.



![](https://img1.daumcdn.net/thumb/R1280x0.fpng/?fname=http://t1.daumcdn.net/brunch/service/user/3K6t/image/_630fSrZQJj7XdswACSoCGDI1vE.png)

사진에서 보는바와 같이 ADD ~ PUSH 까지의 과정을 설명하고 있다. <br><br>

- workspace
  > 현재 내가 작업하는 환경을 말한다 (내 PC) <br>
  > 여기서 작업을 다 마치면 git add 커맨드를 사용할 수 있다.

- staging
  > git add 를 하면 내가 작업한 파일들은 staging 상태가 된다. <br>
  > staging은 대기 상태라고 보면된다. 

- commit 
  > git commit 을 하면 대기상태 에서 내 local repo로 이동시킨다 . <br>
  > local repo는 내가 생성한 저장소이다.

- push
  > git push 를 하면 원격저장소로 지금까지의 수정사항을 반영한다.

- clone/pull
  > clone은 원격 저장소 주소를 가지고 로컬저장소로 복사할 수 있다.
  > pull 은 원격저장소 변경사항 내역을 내 로컬 저장소에 가져온다.
