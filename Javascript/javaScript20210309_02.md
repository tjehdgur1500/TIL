## 객체

### Object

> 객체는 참조타입이다. <br>
> key 와 value 로 이루어져 있고 순서가 없다.


객체생성 방법은 아래처럼 쓰면 된다.

```js

// 객체 생성하기

let myObject = {
    name : 'Seo',
    age : 26,
    favoriteFood : 'rice'
}

```
이것이 객체의 형태이고 , <br>
key : value 한쌍으로 이루어져 있다.<br>
만약 객체의 키에 담긴 값을 알아내려면 이렇게 쓸 수 있다. <br>


- <h1>Dot Notation</h1>
  -  myObject.name >> 따옴표이후 공백이 존재해선 안된다.

- <h1>Bracket  Notation</h1>
  -  myObject[propertyName]  <br>
  -  myObject["propertyName"]  <br>
  -  브라켓 안에는 변수혹은 String 값이 들어간다.
  

```js
myObject["name"]  // 'Seo'
myObject.name // 'Seo'
```
둘의 값은 동일하나 무조건 아무렇게나 쓸 수 있지는 않다.


```js

let myObject = {}

function testFunc(object, nameKey , nameValue){
    object[nameKey] = nameValue;
    return object[nameKey]
}

testFunc(myObject, 'name', 'Seo')
// result : 'Seo'

```

> 알아야할 점 <br>
> 객체의 key 는 string type이다.<br>
> 그렇기 때문에 myObject["name"] 를 했을때 값을 불러올 수 있고 <br>
> myObject[name] 를 했을때는 name이란 변수가 있어야 에러가 나지 않는다.

<br><br>

위의 testFunc 은 각자 다른 방식으로 프로퍼티를 불렀을때의 차이를 보이기 위해 쓴 함수이다. <br>

우선 빈 객체를 초기화 시켰다.<br>
함수의 전달인자로 빈객체와 문자열값을 주었다. <br>
함수에 들어온 각 매개변수들은 객체의 key : value 로 할당시킬 수 있다. <br><br>

하지만 여기서 Dot notation 을 사용한다면 항상 유동적인 값을 바꿀 수 없을것이다. <br>
testFunc(myObject, 'age', 25) <br>
이렇게 새로운 값이 들어가면 dot 는 key 가 생성 되는것이 아닌 <br>
nameKey 라는 변수에 값을 계속 할당하는 것이기에 <br>
Bracket notation을 쓰는게 알맞다 볼 수 있다. <br><br>


Dot notation은 함수를 호출할때 많이 쓰인다. <br><br>
디버그에 자주 쓰이는 console.log() 도 객체와 프로퍼티다 <br><br>

객체에 
```js
let testObject = {
    test : function(keys){
        console.log(keys)
    }
}

testObject.test('Hello')
```
마찬가지로 위와 같이 프로퍼티가 함수가 되는 기능도 만들 수 있다.<br><br>

객체의 모든 키를 불러올 수 있는 for .. in 도 존재한다.

```js
let test = {
    a : 1,
    b : 2,
    c : 3
}

for (let el in test){
    console.log(el); // 모든 key 반환
    console.log(test[el]); // 모든 key의 value 반환
}
```


for in 과 for of 의 차이가 있다. <br><br>

- for ... in <br>
    for in 은 열거 가능한 속성들을 모두 반복한다. 

- for ... of <br>
    for of 는 반복가능한 객체들을 모두 가능하다.
